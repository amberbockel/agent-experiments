# Reading Backlog

**Category:** productivity  
**One-liner:** Track articles, papers, and links you want to reference later with notes on why they matter and tags to find them fast.

## What it does

A simple reading log for Substack writers, researchers, and anyone drowning in browser tabs. Save links with a quick note about why you saved them, tag for later discovery, mark as read when done. Search by title, URL, tags, or notes. Everything persists locally.

## Who it's for

- Amber (Substack research workflow)
- Anyone who saves 47 articles "to read later" and never finds them again
- Writers building a reference library
- Researchers organizing paper trails

## Key features

- **Add links fast** — paste URL, title auto-populates if you want, add tags and notes inline
- **Tag for discovery** — multi-tag support, filter by tag(s)
- **Search everything** — searches title, URL, tags, and notes
- **Mark as read** — simple toggle, filter unread/read/all
- **Minimal UI** — flat iOS base with a journal-like vibe (serif titles, monospace URLs)

## Visual direction

Clean, minimal, almost journal-like. Lots of whitespace. Serif for article titles (Georgia), monospace for URLs (SF Mono), flat iOS otherwise. Light gray background, white cards, generous padding. Think: Apple Notes meets Arc browser's research aesthetic.

## Technical notes

- Single HTML file, ~20KB
- localStorage persistence
- Mobile-first (iPhone with notch)
- Light + dark mode
- No dependencies, no CDN
